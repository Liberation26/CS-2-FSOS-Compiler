# Oryn 0.2.8

## Stage 2 CFG visibility fix

This release keeps the Stage 2 control-flow graph implementation from `0.2.7`, but makes the CFG proof lines visible during the normal compiler/run flow.

Changes:

- The compiler now prints `[ OK ] [ CFG      ]` lines to standard output after IR lowering.
- `Runqemu.sh` now prints the CFG proof lines from the compiler diagnostics log before continuing to native build/QEMU.
- The Stage 2 compiler test now checks both the diagnostics log and compiler standard output for visible CFG proof lines.
- Bumped `Runqemu.sh` to version `0.2.8`.

The CFG data is still written to the `.stage2.ir.json` manifest and the diagnostics log. This fix makes it clear on-screen that Stage 2 generated the control-flow graph.
