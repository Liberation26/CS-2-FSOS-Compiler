# Oryn 0.2.15

Stage 2 now includes a dedicated test kernel under `OSes/Stage2/`.

## Added

- `OSes/Stage2/README.md` documenting the Stage 2 runtime proof.
- `OSes/Stage2/Source/Kernel.cs` now proves:
  - `Diagnostics.WriteOk` calls,
  - `Memory.Initialize`,
  - integer local variables,
  - arithmetic,
  - `while`,
  - `if` / `else`,
  - static helper methods,
  - and `Cpu.HaltForever`.

## Changed

- `Runqemu.sh` version bumped to 0.2.15.
- `Runqemu.sh` now keeps Stage 2 development mode focused on the second kernel; `All` runs Stage 2 only, and Stage 1 selectors are rejected with a clear message.
- Stage 2 compiler proof checks now verify the runtime proof string literals used by the dedicated test kernel.

## Expected Stage 2 runtime proof

The Stage 2 kernel prints repeated string-only proof lines because diagnostics formatting is not available yet:

```text
[ OK ] Stage2 kernel entered
[ OK ] Stage2 memory initialized
[ OK ] Stage2 loop tick
[ OK ] Stage2 loop tick
[ OK ] Stage2 loop tick
[ OK ] Stage2 branch worked
[ OK ] Stage2 helper method worked
[ OK ] Stage2 kernel is halting forever
```
