# Oryn 0.1.1

## Added

- Added root `Runqemu.sh` as the first end-to-end freestanding backend/QEMU proof script.
- `Runqemu.sh` runs `Oryn.Compiler`, generates a Stage 1 backend assembly file, creates a small x86_64 freestanding boot harness, links an ELF64 kernel, and launches it in `qemu-system-x86_64`.
- Added serial/VGA diagnostics output for the QEMU proof path so `Diagnostics.WriteOk`, `Diagnostics.WriteWarn`, and `Diagnostics.WriteFail` produce visible kernel proof lines.

## Changed

- Bumped the compiler, updater, archive, and repository version to `0.1.1`.
- `update.sh` now calls `Runqemu.sh` after applying the update, committing, and attempting to push.
- `update.sh` also calls `Runqemu.sh` when there are no file changes to commit, because the update step has still completed.

## Notes

- The Stage 1 `.o` emitted by `Oryn.Compiler` remains a placeholder; `Runqemu.sh` uses the generated assembly backend as the first runnable backend proof.
- The QEMU run intentionally times out successfully because the generated kernel reaches `Cpu.HaltForever()`.
