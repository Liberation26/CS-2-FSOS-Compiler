# Oryn 0.9.0 Release Notes

## Stage 9: generated kernel template composition from selected modules

Oryn 0.9.0 adds the Stage 9 kernel composition step. The compiler can now generate a safe C# kernel source file from a template and the selected module manifests before the normal compile/backend path runs.

## Added

- `compose-kernel` compiler command.
- `Source/Core/Oryn.Compiler/KernelComposition/KernelTemplateComposer.cs`.
- `OSes/Stage9/Templates/Kernel.template.cs`.
- `OSes/Stage9/README.md`.
- `Tests/Compiler/Stage9/` test suite.
- Stage 9 support in `Runqemu.sh`.

## What Stage 9 proves

Stage 9 proves that Oryn can:

- resolve selected module manifests and their dependencies,
- generate C# kernel source from a template,
- insert deterministic diagnostics proving the selected modules,
- validate the generated source before backend/native compilation,
- reject invalid calls before generated C, generated assembly, or ELF64 object files are produced,
- boot the generated kernel through the existing manifest/API-contract runtime path.

## Invalid-call proof

`Tests/Compiler/Stage9/03-invalid-call-fails-before-native-compile.sh` compiles a kernel containing `Diagnostics.SecretWrite(...)`. That call is not in the approved binding/API-contract catalogue. The test requires the compiler to fail and confirms that no object, generated C, or generated assembly artifact was created.

## Running

```bash
ORYN_BUILD_COMPILER=1 ./Runqemu.sh Stage9
./Tests/Compiler/Stage9/run.sh
```
