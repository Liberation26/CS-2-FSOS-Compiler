# Oryn 2.0.3 Release Notes

Oryn 2.0.3 improves the visual configurator so it behaves like a forms-based project configuration app instead of a repeated question prompt.

## Fixed

- The visual configurator no longer forces reconfiguration only because an older OS project is missing the `VisualConfiguratorCompleted` marker.
- Existing OS projects reuse saved answers unless a required answer is genuinely missing or invalid.
- The new-OS form now guesses sensible defaults from the current directory and current question files.
- OS Name and Kernel Name are auto-suggested from OS Title and remain strict no-space identifiers.

## Changed

- `OrynVisualConfigurator` now presents one editable form with pre-filled values.
- Choice questions remain drop-downs or check boxes from `Questions/*.question.json`.
- Free text remains limited to OS Title, OS Name, and Kernel Name.
- Other non-choice fields are rendered read-only unless they are one of the approved user text fields.
