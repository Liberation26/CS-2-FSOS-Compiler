# Oryn 0.3.1

## Purpose

Correct the Stage 3 QEMU proof harness after the direct ELF64 object writer kernel booted successfully but Runqemu still checked for Stage 2 proof strings.

## Changed

- Updated `Runqemu.sh` to validate `${STAGE_NAME}` kernel entry and halt diagnostics.
- Updated the Stage 3 QEMU test to require exact Stage 3 proof lines.
- Bumped package metadata and script versions to `0.3.1`.

## Proof expected

A Stage 3 run is successful when QEMU reaches:

```text
[ OK ] [ KERNEL   ] Stage3 kernel entered
[ OK ] [ KERNEL   ] Stage3 memory initialized
[ OK ] [ KERNEL   ] Stage3 loop tick
[ OK ] [ KERNEL   ] Stage3 branch worked
[ OK ] [ KERNEL   ] Stage3 helper method worked
[ OK ] [ KERNEL   ] Stage3 kernel is halting forever
```

The QEMU timeout after the halt loop is still treated as success.
