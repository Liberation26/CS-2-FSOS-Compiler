# Oryn 0.3.3 Release Notes

## Stage 3 verification and object inspection

Oryn 0.3.3 adds a Stage 3 verification pack for the direct ELF64 object writer.

Stage 3 already proved that Oryn can emit a bootable kernel object directly. This release adds structural inspection so the generated `Kernel.stage3.o` is checked as an object file, not only as part of a successful boot.

## Added

- `Tests/Compiler/Stage3/04-elf64-header-check.sh`
- `Tests/Compiler/Stage3/05-elf64-section-check.sh`
- `Tests/Compiler/Stage3/06-elf64-symbol-check.sh`
- `Tests/Compiler/Stage3/07-elf64-relocation-check.sh`
- `Tests/Compiler/Stage3/08-stage3-no-compiler-rebuild-check.sh`

## Verified by the new tests

The new Stage 3 tests check:

- ELF magic
- ELF64 class
- little-endian encoding
- `ET_REL` relocatable object type
- `EM_X86_64` machine type
- valid section header table
- required sections:
  - `.text`
  - `.rodata`
  - `.rela.text`
  - `.symtab`
  - `.strtab`
  - `.shstrtab`
  - `.note.GNU-stack`
- defined global function symbols:
  - `Kernel_Main`
  - `Kernel_WriteBanner`
- unresolved global external symbols for approved module/runtime calls:
  - `Diagnostics_WriteOk`
  - `Diagnostics_WriteFail`
  - `Memory_Initialize`
  - `Cpu_HaltForever`
- local `.rodata` symbols for string literals
- relocation entries targeting valid symbols
- currently supported relocation kinds:
  - `R_X86_64_PC32`
  - `R_X86_64_PLT32`
- Stage 3 tests do not rebuild the compiler unless `ORYN_BUILD_COMPILER=1` is explicitly set

## Build policy

This release preserves the existing rule that `Runqemu.sh` and the Stage 3 tests use the existing compiler DLL by default.

To rebuild the compiler deliberately:

```sh
ORYN_BUILD_COMPILER=1 ./Tests/Compiler/Stage3/run.sh
```
