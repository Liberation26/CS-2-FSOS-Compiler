# Oryn 0.2.6

## Stage 2 Phase 3: Real Oryn IR

Adds a proper, explicit Stage 2 intermediate representation. The compiler now parses a small statement/expression model instead of only call statements, semantically binds locals and approved module calls, and lowers code into stack-style Oryn IR instructions.

Implemented minimum IR operations:

- `Label`
- `Jump`
- `JumpIfFalse`
- `Return`
- `Call`
- `DeclareLocal`
- `LoadLocal`
- `StoreLocal`
- `ConstInt32`
- `ConstString`
- `AddInt32`
- `SubInt32`
- `CompareEqualInt32`
- `CompareLessThanInt32`

The parser now accepts simple `int` locals, assignments, `if` blocks, `else` blocks, `while` loops, `return;`, arithmetic expressions, comparison expressions, and approved module calls such as `Diagnostics.WriteOk("Done");`.

The backend manifest is now written as `<output>.stage2.ir.json` to make the IR output clear. The generated assembly and C backend snippets consume the new IR stream.
