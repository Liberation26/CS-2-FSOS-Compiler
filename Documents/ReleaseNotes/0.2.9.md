# Oryn 0.2.9

Stage 2 Phase 5 adds the real x64 backend path.

- Emits clang/as-compatible x64 assembly from Oryn IR.
- Uses stack-frame backed local variables instead of global local placeholders.
- Keeps explicit label, jump, conditional jump, arithmetic, comparison, call, and return lowering.
- Updates Runqemu and compiler messages to treat `.generated.S` as the real Stage 2 kernel body.
- Leaves direct ELF64 object writing to Stage 3.
