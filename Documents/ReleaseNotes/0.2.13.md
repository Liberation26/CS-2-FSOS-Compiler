# Oryn 0.2.13

## Stage 2 - Static helper methods

This release adds Stage 2 support for private/public static helper methods on the generated `Kernel` class. User code can now split work out of `Main()` without needing to flatten everything into one method.

Implemented:

- Parses all `public static void` and `private static void` methods in `Kernel`.
- Resolves unqualified helper calls such as `WriteBanner();` as `Kernel.WriteBanner`.
- Lowers helper calls to native `Kernel_*` symbols, for example `Kernel_WriteBanner`.
- Emits one x64 function body per Stage 2 method, each with its own rbp stack frame and local-slot table.
- Keeps approved module calls such as `Diagnostics.WriteOk(...)` and `Cpu.HaltForever()` routed through the existing binding catalogue.
- Extends the Stage 2 proof test to check `Kernel_Main`, `Kernel_WriteBanner`, and `call Kernel_WriteBanner`.

The direct ELF64 object writer remains a Stage 3 task; Stage 2 still assembles generated `.S` through clang/as.
