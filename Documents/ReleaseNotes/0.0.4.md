# Oryn 0.0.4 Release Notes

## Update workflow fixes

- `update.sh` now selects the highest semantic `Oryn-x.y.z.zip` version from the downloads directory when no archive path is supplied.
- `update.sh` now compares itself with the updater inside the selected archive.
- When the archive contains a different `update.sh`, the running updater replaces itself and restarts before applying `ChangedFiles/`.
- The updater continues to display its own version number at startup.
