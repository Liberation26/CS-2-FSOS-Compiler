# Oryn 0.6.1

Stage 6.1 fixes the silent Stage 6 QEMU boot proof failure.

## Fixed

- Enabled GRUB serial output in the generated ISO boot configuration so loader errors are visible in QEMU logs.
- Switched the generated GRUB entry to `multiboot2 /boot/OrynKernel.elf` for the Stage 6 ISO boot path.
- Added a native pre-kernel serial proof line before `Kernel_Main` is called.
- Fixed generated manifest glue so `ManifestLoader` does not recursively call `ModuleManifest_InitializeSelected`.
- Added idempotence to `ModuleManifest_InitializeSelected`.
- Updated Stage 6 proof checks to require the native handoff line, ManifestLoader glue activation, Runtime initialization, Memory initialization, and manifest runtime completion.

## Expected Stage 6 proof

A successful Stage 6 run now proves the path reaches GRUB, enters the native boot stub, reaches long mode, calls the generated kernel, initializes manifest-selected modules, and halts intentionally.
