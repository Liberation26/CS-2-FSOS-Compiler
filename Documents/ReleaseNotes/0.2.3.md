# Oryn 0.2.3

## Stage 2, phase 2: proper frontend separation

This release moves compiler logic out of `Program.cs` and into real compiler components.

## Changed

- `Program.cs` is now a small entry point that reports the compiler version and delegates to the CLI.
- Added `CompilerCli`, `CompilerCommand`, `CompilerPipeline`, and `CompilerResult` as the host-side command/pipeline layer.
- Added frontend parser and safe-subset validator components under `Frontend/`.
- Added semantic binding through `Frontend/SemanticModel/` and symbol resolution through `KernelModel/SymbolTable/`.
- Added kernel AST records under `KernelModel/KernelAst/`.
- Added Oryn IR and a first control-flow graph model under `IR/`.
- Added native x64 backend emitters under `Backends/Native/X64/`.
- Added object-placeholder backend output under `Backends/Native/Object/`.

## Notes

The generated `.o` file is still a text placeholder. The generated `.generated.S` assembly is still the backend proof used by `Runqemu.sh` to build the freestanding kernel.
