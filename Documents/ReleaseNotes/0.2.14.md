# Oryn 0.2.14

Stage 2 now loads approved module bindings from JSON instead of hardcoding the starter module map inside compiler source.

Implemented:

- Added JSON-loaded binding catalogue support for `Source/Sdk/Bindings/*.binding.json`.
- Kept starter binding files for Diagnostics, Cpu, and Memory as the source of truth.
- Added explicit `argumentCount` values to binding JSON records.
- Removed the built-in `Diagnostics.WriteOk`, `Memory.Initialize`, and `Cpu.HaltForever` binding table from `BindingCatalog`.
- Updated generated C declarations to be derived from IR native calls rather than a fixed list of starter modules.
- Kept Stage 2 x64 generation compatible with JSON-loaded native symbols.
- Updated `Runqemu.sh` so the default run path and legacy `All` path run only the Stage 2 kernel during this compiler phase.

This means adding an approved module now follows the intended expandable path: add the API DLL, add binding JSON, add the native implementation, and add tests, without editing compiler binding source manually.
