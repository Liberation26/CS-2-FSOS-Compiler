# Oryn 0.2.1

## Purpose

Fix the first Stage 2 run feedback. The previous 0.2.0 package could build and boot the Stage2 ISO, but the default runner only ran one selected stage and the early serial proof could be missed.

## Changes

- Bumped Oryn to 0.2.1.
- Updated `Runqemu.sh` to default to `All`.
- Added `./Runqemu.sh All` support.
- Kept `./Runqemu.sh Stage1` and `./Runqemu.sh Stage2` for single-stage regression runs.
- Added a boot-level serial proof line immediately after entering long mode and before calling `Kernel_Main`.
- Updated native diagnostics serial output to wait for the COM1 transmitter-ready bit before writing each byte.
- Updated documentation for the new stage running behaviour.

## Expected proof

A default run should now execute Stage1 and then Stage2. During each boot, serial output should include the boot proof line followed by kernel diagnostics.
