# Oryn 0.1.0 Release Notes

Oryn 0.1.0 introduces the first working compiler and backend proof.

## Added

- Implemented `oryn compiler compile <source.cs> --target x64-elf --output <output.o>`.
- Added Stage 1 safe-subset validation for the initial kernel source proof.
- Added parsing for `public static void Main()` containing approved module API calls.
- Added lowering from approved SDK calls to native symbols:
  - `Diagnostics.WriteOk(...)` → `Diagnostics_WriteOk`
  - `Diagnostics.WriteWarn(...)` → `Diagnostics_WriteWarn`
  - `Diagnostics.WriteFail(...)` → `Diagnostics_WriteFail`
  - `Memory.Initialize()` → `Memory_Initialize`
  - `Cpu.HaltForever()` → `Cpu_HaltForever`
- Added backend output generation:
  - `<output>.stage1.json`
  - `<output>.generated.c`
  - `<output>.generated.S`
  - `<output>` placeholder for the future ELF64 object writer

## Notes

The `.o` output is not yet a linkable ELF64 relocatable file. This release proves the front-end command, safe-subset gate, call lowering, IR manifest, and first backend text emitters. The next compiler backend milestone is the real ELF64 relocatable object writer.
