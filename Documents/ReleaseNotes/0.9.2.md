# Oryn 0.9.2 Release Notes

## Purpose

Oryn 0.9.2 fixes the Stage 9 execution path after 0.9.1.

Stage 9 added generated kernel template composition through the `compose-kernel` compiler command. The root `Runqemu.sh` selected Stage9 correctly, but it reused an already-built compiler DLL by default. On an existing tree that DLL could still be an older compiler, such as 0.2.16, which caused:

```text
[FAIL] Unknown command: compose-kernel
```

## Changes

- Bumped Oryn to 0.9.2.
- Bumped `Runqemu.sh` to 0.9.2.
- Bumped `Oryn.Compiler` reported version to 0.9.2.
- Changed root `Runqemu.sh` so `ORYN_BUILD_COMPILER` defaults to `1`.
- Stage9 now rebuilds `Source/Core/Oryn.Compiler/Oryn.Compiler.csproj` before running `compose-kernel`, unless explicitly overridden with `ORYN_BUILD_COMPILER=0`.

## Proof intent

This release ensures the Stage9 composition path uses the source tree just applied by `update.sh`, so the `compose-kernel` command is present before the generated kernel is composed and before backend/native compilation begins.
