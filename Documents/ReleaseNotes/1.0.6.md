# Oryn 1.0.6 release notes

Oryn 1.0.6 fixes the end-user module selection flow after the generated OS boot path was proven in 1.0.5.

## Fixed

- Added an explicit `None` option for additional user-selected modules.
- Changed the generator default additional module selection from `Memory` to `None`.
- `None` now produces an empty `UserSelectedModules` array in the generated answers and manifest.
- `None` cannot be combined with other optional modules.
- `Runqemu.sh` now always passes the mandatory module list to kernel composition, even when no optional modules are selected, preventing the composer from falling back to all link-by-default modules.

## Policy

Mandatory kernel modules remain separate from user-selected modules. Runtime, Diagnostics, Panic, Cpu, and ManifestLoader are linked automatically. Diagnostics and Panic are always enabled.
