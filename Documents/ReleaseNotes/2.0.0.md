# Oryn 2.0.0 Release Notes

Oryn 2.0.0 introduces the visual-first project configuration model.

## Added

- `Applications/OrynVisualConfigurator/`.
- Visual-first OS creation and configuration flow.
- Project search and manual directory load support.
- Current-directory OS project detection.
- `OS Title` as a human-facing display value.
- Strict `OS Name` and `Kernel Name` validation with no spaces.
- Saved answer/profile reuse for build and run.
- Automatic configurator launch only when an OS has not been configured visually or has missing required questions.
- Rewritten README for the 2.0.0 workflow.

## Changed

- `./Oryn.sh new` now launches `OrynVisualConfigurator`.
- `./Oryn.sh configure [path]` opens the visual configurator.
- `./Oryn.sh configure --search` searches generated OS projects.
- `./Oryn.sh configure --load` prompts for a directory.
- `./Oryn.sh build` and `./Oryn.sh run` can infer the OS project from the current working directory.
- Question files are the source of truth for the visual configurator.

## Policy

- The configurator does not hardcode the questions for the version.
- Known-choice questions are shown as selectable choices.
- Free text is limited to OS title/name and kernel name.
- OS name and kernel name must be path/symbol-safe identifiers.
