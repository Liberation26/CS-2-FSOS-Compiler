# Oryn 0.2.4 Release Notes

## Purpose

This is a follow-up fix for Stage 2, phase 2 after `update.sh` appeared to stop at:

```text
[ OK ] [ RUNQEMU  ] Building Oryn.Compiler once for all requested stages
```

## Fixed

- `Runqemu.sh` no longer hides `dotnet build` output.
- Compiler build output is now shown live and written to `Build/Oryn.Compiler.build.log`.
- Compiler builds use `--disable-build-servers` to avoid stale MSBuild build-server hangs.
- Compiler builds are guarded by `ORYN_COMPILER_BUILD_TIMEOUT`, defaulting to 240 seconds.
- If the compiler build fails or times out, `Runqemu.sh` now prints a clear `[FAIL]` line and points to the build log.

## Notes

The Stage 2 frontend separation from 0.2.3 remains in place. This release only fixes the run/update visibility and hang diagnostics around the compiler build step.
