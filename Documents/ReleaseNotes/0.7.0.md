# Oryn 0.7.0 Release Notes

## Stage 7: module dependency graph proof

This release adds Stage 7, which proves that Oryn can resolve selected module manifests by dependency graph rather than relying on a manually correct list order.

## Added

- `OSes/Stage7/` dependency-resolved kernel proof.
- `Source/Core/Oryn.Compiler/Manifests/ModuleDependencyResolver.cs`.
- `DependsOn` support in `Source/Sdk/ModuleManifests/*.module.json`.
- Stage 7 manifest glue generation in `Runqemu.sh`.
- Stage 7 tests under `Tests/Compiler/Stage7/`.

## Proven

- Selected module manifests are loaded.
- Required dependencies must also be selected.
- Missing dependencies are rejected.
- Circular dependencies are rejected.
- Initialization order is resolved from the dependency graph.
- Only the resolved selected native module sources are compiled and linked.
- Build-time diagnostics and QEMU runtime output agree on the resolved order.
