# Oryn 0.1.5

## Fix

The Stage 1 compiler and QEMU backend now produce diagnostics in two places.

- `Oryn.Compiler compile` writes `Kernel.stage1.diagnostics.log` beside the generated Stage 1 backend files.
- The diagnostics log records every lowered call and explicitly notes which `Diagnostics.Write*` calls will be emitted by the freestanding kernel.
- The native Diagnostics module now writes DEBUG output to QEMU serial and VGA text memory instead of discarding messages.
- `Runqemu.sh` now verifies that the compiler diagnostics log exists before compiling/linking the kernel.

## Expected proof lines

When QEMU runs the Stage 1 kernel, the generated kernel should emit lines like:

```text
[ OK ] [ KERNEL   ] Stage1 kernel entered
[ OK ] [ KERNEL   ] Stage1 memory module initialized
[ OK ] [ KERNEL   ] Stage1 kernel is halting forever
```
