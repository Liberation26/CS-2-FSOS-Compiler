# Oryn 0.8.0 Release Notes

## Stage 8: module API contracts / approved calls from C#

Oryn 0.8.0 adds explicit module API contracts for approved calls made from Oryn-safe C#.

## Added

- `Source/Sdk/ApiContracts/` containing one API contract JSON file per approved module.
- `ModuleApiContractCatalog` in the compiler frontend.
- Stage 8 semantic validation that requires each approved C# module call to match both the binding catalogue and the API contract catalogue.
- `OSes/Stage8/` proof kernel.
- `Tests/Compiler/Stage8/` test suite.
- Stage 8 support in `Runqemu.sh`.

## What Stage 8 proves

Stage 8 proves that an exposed C# call is not approved merely because a native symbol exists. The compiler now requires an explicit contract that matches:

- module name,
- namespace,
- type name,
- managed method name,
- native symbol,
- argument count,
- argument types,
- approval for C# kernel use.

## Expected result

The Stage 8 kernel boots through the Stage 7 dependency-safe module graph, validates contract-approved calls, initializes selected modules, marks the runtime ready, and halts forever.
