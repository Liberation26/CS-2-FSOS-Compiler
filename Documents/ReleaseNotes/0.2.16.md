# Oryn 0.2.16

Stage 2 boot/output fix package.

## Fixed

- Fixed the Stage 2 x64 backend so each native or helper call is emitted once instead of twice.
- Updated the Stage 2 GRUB ISO boot entry to use the Multiboot v1 header path, matching the boot header emitted by the freestanding kernel image.
- Added a QEMU debugcon output path on port `0xE9` as a secondary boot proof channel when the serial file is unexpectedly empty.
- Mirrored early boot serial proof output to debugcon before and after long mode entry.
- Mirrored native diagnostics output to debugcon as well as COM1 serial and VGA.
- Updated `Runqemu.sh` to report debugcon output if the serial log is empty rather than silently losing proof output.

## Still true

- Stage 2 remains focused on the second kernel only.
- Stage 2 keeps the stack/local model, string literal table, static helper methods, and JSON module bindings.
