# Oryn 0.1.6

## Fixes

- Changed `Runqemu.sh` to boot the generated x86_64 freestanding kernel through a GRUB ISO instead of passing the ELF directly to QEMU with `-kernel`.
- Added a Multiboot2 header to the generated boot assembly so GRUB can load the 64-bit kernel entry path.
- Added explicit ISO build output under `OSes/Stage1/Build/Runqemu/IsoRoot`.
- Added clearer status lines before and after the compiler run so long .NET first-run/build pauses do not look silent.

## Reason

QEMU's direct `-kernel` path rejected the ELF64 kernel with:

```text
qemu-system-x86_64: Cannot load x86-64 image, give a 32bit one.
```

The kernel is an x86_64 image. The failure was the boot path, not the target architecture.
