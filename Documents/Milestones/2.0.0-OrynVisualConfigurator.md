# Oryn 2.0.0 - OrynVisualConfigurator Milestone

Oryn 2.0.0 turns OS generation into a project-profile workflow.

The central feature is `Applications/OrynVisualConfigurator`, which loads the current version's JSON question files and renders all available questions and choices.

## Goals

- Visual-first project configuration.
- No repeated questions after an OS has saved answers.
- Automatic reconfiguration only when required answers are missing.
- Search and load support for generated OS directories.
- Separate human-facing OS title from strict OS/project identifiers.

## Project detection

The configurator can open projects from:

- current working directory
- `OSes/<Name>`
- explicit relative path
- explicit absolute path
- search under `OSes/`
- manual directory load

## Answer model

`manifest.json` and `Answers/*.answers.json` are the saved generation profile for an OS. Build and run use those answers directly.
