# Oryn 0.2.2

## Fixes

- Changed `Runqemu.sh` to run QEMU headless by default.
- Captured QEMU serial output to `Qemu.serial.log` for each stage and printed the captured log after QEMU exits or times out.
- Added an early 32-bit boot serial proof before the long-mode transition, so a missing serial log is detected as a real failure instead of being treated as a successful halt.
- Kept the existing 64-bit boot serial proof before `Kernel_Main`.
- Changed the all-stage run to build `Oryn.Compiler` once, then execute the compiled compiler DLL for Stage1 and Stage2. This avoids repeated implicit `dotnet run` builds for each kernel.

## Expected default command

```bash
./Runqemu.sh
```

The default now runs:

```text
Stage1
Stage2
```

in headless mode, with serial proof output printed back to the terminal.
