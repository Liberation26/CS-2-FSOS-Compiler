# Oryn 0.2.12

## Stage 2 - String literal table

This release adds the Stage 2 x64 string literal table.

- Emits each distinct Stage 2 string literal once in `.section .rodata`.
- Uses stable assembler-local labels such as `.Lstr0`, `.Lstr1`, and `.Lstr2`.
- Lowers approved one-argument string calls to direct RIP-relative argument loads, for example `lea .Lstr0(%rip), %rdi` followed by `call Diagnostics_WriteOk`.
- Keeps the existing Phase 6 rbp-based stack/local model for integer locals.
- Extends the Stage 2 compiler proof test to check `.rodata`, `.asciz`, `.LstrN`, RIP-relative literal loads, and diagnostics calls.
- Bumped Oryn, Runqemu, and update workflow versions to 0.2.12.
