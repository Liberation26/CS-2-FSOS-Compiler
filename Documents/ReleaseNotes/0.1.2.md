# Oryn 0.1.2 Release Notes

## Purpose

This release fixes the first `Runqemu.sh` integration problem found after `update.sh` launched the freestanding kernel run.

## Fixed

- `Oryn.Compiler.csproj` now excludes `Source/Core/Oryn.Compiler/Tests/**/*.cs` from normal .NET compilation.
- Stage test kernels remain present as compiler input files, but they are no longer compiled as part of `Oryn.Compiler` itself.
- This fixes the build failure where `Kernel.stage0.cs` tried to import `Oryn.Kernel.Diagnostics` and `Oryn.Kernel.Cpu` as if they were .NET project references.

## Changed

- Bumped root `VERSION` to `0.1.2`.
- Bumped `update.sh` version to `0.1.2`.
- Bumped `Runqemu.sh` version to `0.1.2`.
- Bumped `Oryn.Compiler` version output to `0.1.2`.

## Expected result

`update.sh` should now be able to call `Runqemu.sh`, and `Runqemu.sh` should be able to run `dotnet run --project Source/Core/Oryn.Compiler/Oryn.Compiler.csproj -- compile ...` without the Stage0 kernel sample being compiled into the compiler project.
