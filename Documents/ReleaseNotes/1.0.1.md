# Oryn 1.0.1 Release Notes

Oryn 1.0.1 fixes the first end-user generation milestone workflow.

## Fixed

- `Oryn.Generator generate` now visibly asks the end-user questions when values are not supplied on the command line.
- `update.sh` now launches the end-user generation workflow instead of silently running the Stage9 proof kernel by default.
- Mandatory kernel modules remain hidden from user selection.
- Diagnostics and Panic remain always enabled.
- QEMU launch now captures stderr separately and uses an explicit CD-ROM drive path for ISO booting.
- Runqemu validates the generated ISO contents before launching QEMU.

## Policy

User-selected modules exclude the mandatory modules needed to get the kernel running. The mandatory module closure is linked automatically.
